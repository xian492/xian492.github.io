4-4 磁盘文件最优存储

题目

算法实现/解题思路

  • 排序文件概率：
    • 使用 sort 函数对文件的检索概率数组 p 进行排序，以确保概率从小到大排列。
  • 确定文件分布：
    • 设定一个数组 x 用于表示文件在磁盘中的分布情况。
    • 将概率最大的文件 p[n-1] 放置在中心位置 x[k]，其中 k = (n-1)/2 作为中心索引。
    • 接下来，按照指定规则将其他文件放置在磁盘上，满足 f(n-1) 占中心磁道，f(n-2) 和 f(n-3) 分居 f(n-1) 两侧，f(n-4) 和 f(n-5) 分居最外两侧。
  • 计算最小期望检索时间：
    • 对于每个文件位置 i，计算与其他文件位置 j 之间的期望检索时间。
    • 通过计算 x[i]*x[j]*(j-i)，表示两个文件在磁盘上的距离乘以它们的概率乘积。
    • 累加所有文件对的期望检索时间，并累加它们的概率 P = p[1]+p[2]+..+p[n]。
  • 返回最小期望检索时间：
    • 最后将总期望检索时间 sum 除以概率总和 P 的平方，即 sum/P/P，得到最终的最小期望检索时间。

程序实现

复杂度分析

时间复杂度分析：

  • 读取输入和初始化数组 p 的时间复杂度是线性的：**O(n)**，其中 n 是文件的数量。
  • 使用 Arrays.sort(p) 对概率数组进行排序：排序算法的时间复杂度为 **O(n log n)**，这是最耗时的部分。

​ 接下来的循环操作的时间复杂度如下：

  • 初始化数组 x 的时间复杂度是线性的：**O(n)**。
  • 分配值给 x[i] 的三个循环的总时间复杂度为 **O(n)**。
  • 计算 sum 的嵌套循环涉及两层迭代：
    • 外部循环运行 n 次。
    • 内部循环运行 n - i 次，其中 i 是外部循环的当前迭代次数。

因此，greedy 方法的总体时间复杂度可以估算为：**O(n log n)**（排序）+ **O(n)**（数组初始化）+ **O(n)**（值分配）+ **O(n^2)**（嵌套循环）

这段代码的主要耗时部分是排序操作，因此整体的时间复杂度为 **O(n log n)**。

输出结果

  1. 基本输入：

    • 输入：n = 5，p = [1, 2, 3, 4, 5]
    • 输出：0.613333
    • 用例分析：简单的顺序排列情况。

  2. 特殊情况：

    • 输入：n = 1，p = [100]
    • 输出：0.000000
    • 用例分析：单个元素的情况。

  3. 重复元素：

    • 输入：n = 6，p = [5, 5, 5, 6, 6, 6]
    • 输出：0.940312
    • 用例分析：包含重复元素的情况。

  4. 随机输入：

    • 输入：n = 5，p = [33, 55, 22, 11, 9]

    • 输出：0.547396

    • 用例分析：随机生成的一组数据。

结果分析

  1. 基本输入：
    • 输入是简单的顺序排列情况，文件概率按顺序增加。输出值为 0.613333，这是按照中心磁道与两侧文件位置关系计算得出的最小期望检索时间。因为文件顺序是增加的，检索的期望时间相对较小。
  2. 特殊情况：
    • 对于只有一个文件的情况，输出为 0.000000。这个结果意味着贪心算法在这种情况下计算得到的最小期望检索时间为零。因为只有一个文件，不存在多个文件之间的排列顺序，所以检索时间为零是符合预期的。
  3. 重复元素：
    • 输入包含重复元素的情况下，输出为 0.940312。这种情况下，贪心算法按照文件概率大小排序，然后根据一定规则安排文件在磁盘上的位置。尽管有重复元素，但算法仍会按照概率大小顺序进行排序，并计算出相对较小的期望检索时间。
  4. 随机输入：
    • 对于随机生成的输入情况，输出为 0.547396。在这种情况下，贪心算法对随机概率的文件按照规则进行排序，并计算出相对较小的期望检索时间。这展示了算法可以在不同类型的输入情况下产生合理的输出结果。

4-9 虚拟汽车加油

题目

算法实现/解题思路

处理逻辑：

  • 对于每个测试用例，首先从输入中读取两个整数n和k，分别表示目的地距离和加油站数量。
  • 接下来按顺序读取k+1个整数，并将它们存储在数组a中，这些数字表示每个加油站距离起点的距离。

贪心算法处理逻辑：

  • 使用贪心算法求解汽车加油问题。
  • 代码中的 greedy() 方法模拟了汽车行驶过程中的加油决策：
    • 通过一系列循环遍历加油站距离数组a。
    • 使用gas表示当前的油还可以行驶多少距离。
    • 如果出现两个加油站之间的距离大于目的地距离n，则肯定到达不了目的地，将结果设为 -1。
    • 如果当前的油不够行驶到下一个加油站，就加油，并将加油次数sum加一。

输出结果：

  • 最后将计算得到的结果输出到 “output9.txt” 文件中，如果无解则输出 “No Solution”。

程序实现

复杂度分析

时间复杂度分析：

  1. 文件读取操作：
    • 文件读取的时间复杂度是 O(n)，其中 n 是输入文件中测试用例的数量。
  2. 循环遍历操作：
    • 对于每个测试用例，首先会从输入中读取两个整数 n 和 k，然后读取 k+1 个整数，因此循环遍历的时间复杂度是 O(k)。
  3. 贪心算法处理逻辑：
    • 贪心算法的时间复杂度主要集中在循环遍历加油站距离数组 a 的操作上。
    • 算法会遍历 k+1 个加油站距离，对于每个加油站进行一些判断和计算，因此这部分的时间复杂度也是 O(k)。

整体来看，这个greedy算法的时间复杂度主要由文件读取和循环遍历操作决定。文件读取部分的复杂度是线性的，而贪心算法部分的复杂度与加油站数量 k 相关，因此这个算法的时间复杂度可以近似表示为 O(n * k)。

如果考虑到 k 的数量级不太大，可以将这个算法的时间复杂度视作近似为 O(n)。

输出结果

基本情况：

  • 加满油后可行驶距离为 70 公里，途中有 3 个加油站，距离分别是 30、40、50 和 60 公里。
  • 输出：2

边界情况：

  • 只有一个加油站，加满油后可行驶距离为 100 公里，加油站与目的地的距离为 110 公里。
  • 输出：No Solution

特殊情况：

  • 加满油后可行驶距离为 30 公里，有 6 个加油站。其中第4个加油站的距离为 0 公里。
  • 输出：2

随机情况：

  • 输出：4

结果分析

  1. 基本情况：
    • 加满油后可行驶距离为 70 公里，途中有 3 个加油站，距离分别是 30、40、50 和 60 公里。
    • 输出结果为 2。这表示汽车需要在第一个和第二个加油站之间加油一次，然后在第三个加油站处加油一次才能到达目的地。
  2. 边界情况：
    • 只有一个加油站，加满油后可行驶距离为 100 公里，加油站与目的地的距离为 110 公里。
    • 输出结果为 No Solution。这意味着汽车无法到达目的地，因为唯一的加油站距离目的地太远，车辆无法到达。
  3. 特殊情况：
    • 加满油后可行驶距离为 30 公里，有 6 个加油站。其中第4个加油站的距离为 0 公里。
    • 输出结果为 2。这表明汽车需要在第一个和第二个加油站之间加油一次，然后在第五个加油站处加油一次才能到达目的地。第四个加油站的存在不影响结果，因为它位于目的地之前。
  4. 随机情况：
    • 加满油后可行驶距离为 7 公里，有 7 个加油站，它们之间的距离分别是 1、2、3、4、5、1 和 6 公里。
    • 输出结果为 4。这表示汽车需要在第一个、第二个、第四个和第六个加油站之间各加一次油才能到达目的地。

这些输出结果显示了在不同情况下算法的执行情况，对于不同的输入，算法给出了符合预期的输出，包括能够到达目的地的最少加油次数和无法到达目的地的情况。

4-11 删数问题

题目

算法实现/解题思路

贪心算法处理逻辑：

  • greedy() 方法实现了贪心算法来找出剩下数字组成的最小数。
  • 如果 k 大于等于 a 的长度，直接返回空字符串，因为数字被删完了。
  • 否则，通过循环进行 k 次删除操作：
    • 在每次循环中，找到第一个数字比后一个数字大的位置 i。
    • 删除数字 a[i]，并将字符串 a 重新赋值为删除后的结果。
  • 最后，检查剩下数字中是否有前导的数字0，如果有，去除前导0，然后返回结果。

程序实现

复杂度分析

时间复杂度分析：

  1. 文件读取操作：
    • 文件读取的时间复杂度是 O(1)，因为它只进行了一次文件读取。
  2. 贪心算法处理逻辑：
    • 首先检查 k 是否大于等于 a 的长度，如果是，则直接返回空字符串，时间复杂度为 O(1)。
    • 否则进入循环，最多进行 k 次迭代。在每次迭代中：
      • 找到第一个数字比后一个数字大的位置 i，这一步最坏情况下需要遍历整个数字串，时间复杂度为 O(n)，其中 n 是数字串 a 的长度。
      • 删除字符并重新构建字符串，这一步操作的时间复杂度为 O(n)，其中 n 是数字串 a 的长度。
    • 前导0的去除时间复杂度为 O(n)，最坏情况下需要遍历整个数字串。

综上所述，贪心算法的时间复杂度主要集中在数字串的遍历和删除操作上，因此整体时间复杂度可近似表示为 O(k * n)，其中 k 是要删除的数字数量，n 是数字串 a 的长度。

输出结果

基本情况：

  • 这里，数字是 174583，需要删除 4 个数字。
  • 输出：13

边界情况：

  • 这个测试用例中，数字是 10200，需要删除 1 个数字。
  • 输出：200

特殊情况：

  • 这个测试用例中，数字是 12345，需要删除 5 个数字。删除的数字数量与整个数字的长度相同，防止整数被完全删除，直接输出 0。
  • 输出：0

随机情况：

  • 这里，数字是一个较长的整数 43210987654321，需要删除 5 个数字。
  • 输出：87654321

结果分析

这些例子显示了在各种情况下算法的表现。它在大多数情况下产生了符合预期的最小可能值。
