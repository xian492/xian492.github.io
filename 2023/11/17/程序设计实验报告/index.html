概述

实验内容

​ 领域特定语言（Domain Specific Language，DSL）可以提供一种相对简单的文法，用于特定领域的业务流程定制。

​ 本作业要求定义一个领域特定脚本语言，这个语言能够描述在线客服机器人（机器人客服是目前提升客服效率 的重要技术，在银行、通信和商务等领域的复杂信息系统中有广泛的应用）的自动应答逻辑，并设计实现一个解释器 解释执行这个脚本，可以根据用户的不同输入，根据脚本的逻辑设计给出相应的应答

基本要求

  1. 脚本语言的语法可以自由定义，只要语义上满足描述客服机器人自动应答逻辑的要求。
  2. 程序输入输出形式不限，可以简化为纯命令行界面。
  3. 应该给出几种不同的脚本范例，对不同脚本范例解释器执行之后会有不同的行为表现。

扩展

  1. 实现了语法检查
  2. 实现了脚本错误类型显示，精确到某一行；
  3. 实现了精良的用户交互体验，直观简洁。

脚本语法

  1. State

    • 脚本分为多个 State

    • 每一个 State 的第一行需要以 State 开头，其后跟一个空格和自定义的 stateId 用于唯一标识 State

    • 每一个 State 的中间不能出现以 State 开头的行，否则会视为两个 State

  2. 字符串

    • 字符串可以分为变量类型和文本类型，可以使用 + 连接为一个字符串

    • 变量类型的字符串使用 $ 符号开头，后面跟随变量名称

    • 文本类型的字符串使用双引号包裹，其中为文本内容

  3. 语句

    • 每一个 State 中拥有多个语句，每一句为一个动作

    • 以 Speak 开头的语句作用是在该 State 被执行时发送消息，在空格后跟随一个字符串

    • 以 Listen 开头的语句作用是指定监听的时间，在空格后跟随一个数字，单位为秒

    • 以 Branch 开头的语句的作用是指定分支，参数为用户的输入和下一个 State 的 id，用逗号分隔，用户的输入为包含匹配

    • 以 Silence 开头的语句的作用是在用户沉默时跳转，参数为跳转到的 State 的 id

    • 以 Default 开头的语句的作用是在以上规则都无法指定下一个 State 时的默认 State

    • 以 Calculate 开头的语句的作用是对变量进行计算，参数以逗号分隔，共三个参数，分别为需要计算的变量，计算完成之后跳转到的 stateId，计算公式，其中计算公式符合 Javascript 语法

    • 以 Exit 开头的语句的作用是标注执行完该 Step 则结束聊天，没有参数

  4. 注释

    • 以 # 开头的行将被忽略

    • 在每一行中，在字符串外的 # 后的内容将被忽略

  5. 规定

    • 每一个脚本中至少需要有一个 State 和一个含有 Exit 语句的 State

    • 每一个不含有 Exit 语句的 State 中都需要有 Default 语句

    • 每一个 stateId 都需要在脚本中被定义

    • 同一个 State 中的每一个 Calculate 都会被执行，stateId 以最后一个 Calculate 的为准

设计和实现

​ 本项目采用纯前端开发模式，允许用户在网页中实时编辑和运行脚本。特别设计了数据结构以保存运行环境和上下文。项目借鉴了PPT的实现方式并进行了优化。首先将脚本语言解析为语法树，然后进行运行，确保了代码的正确性和运行稳定性。

模块划分

DataStore.ts

​ 总线模块，用于存储一些全局信息，管理数据状态的文件，主要负责管理用户编辑的代码以及相关的解析和存储操作，便于在组件间共用

  1. 数据存储和初始化：

    • 定义了 activeCode 和 defaultCode，分别用于存储用户活动的代码和默认的代码。
    • 尝试从 localStorage 中获取用户活动的代码，如果没有，则使用默认代码。

  2. 代码解析：

    • 尝试解析 activeCode，将其转换为抽象语法树（AST）。如果解析失败，则应用默认代码，并打印相关错误信息。

  3. Vue 响应式数据：

    • 使用 reactive 创建了 DataStore 对象，包含了 activeCode、defaultCode、ast、userList 等属性，以响应式地管理这些数据。

  4. 用户代码变化监听：

    • 使用 watch监听 DataStore.activeCode的变化，当活动代码发生变化时，进行相应的操作：

      • 尝试解析新的代码，并更新 DataStore.ast。

      • 如果新代码与旧代码不同，则更新 localStorage 中的 activeCode，并打印代码应用成功的信息。

      • 如果解析失败，则将 activeCode 回滚到旧值，并打印相关错误信息。

interface.ts

​ 是类型定义模块，用于规定和注释自定义的类型，便于编码时使用。文件定义了用于描述聊天机器人交互过程中各种类型和结构的 TypeScript 接口。

parse.ts

​ 是解析脚本的模块，输入脚本，输出语法树

包含函数：

  1. parse:
    • 这是入口函数，用于解析输入的代码字符串并生成语法分析树。
    • 将代码字符串按行解析，忽略空行和注释，对每一行调用 parseLine 函数解析。
  2. parseLine:
    • 用于解析代码的每一行。
    • 去除空格，忽略注释，并根据空格分割出不同的 token。
    • 根据 token 类型调用相应的处理函数。
  3. 处理函数（processState, processSpeak, processListen, processBranch, processSilence, processDefault, processCalculate, processExit）:
    • 每个处理函数针对不同类型的 token 执行特定的操作，以构建语法分析树的结构。
    • 对应的处理函数会创建或更新 AST 中的相关信息，如 state、speak、listen、branch、silence、default、calculate 等。
  4. validate:
    • 用于验证生成的语法分析树的正确性。
    • 确保至少有一个 state，并检查 exit 列表、default 状态、silence 状态、listen 时间和 branch 状态是否有效。

实现思路：

  1. 分割代码：
    • 通过 split 函数将代码分割为行列表。
    • 对每一行进行处理，忽略空行和注释。
  2. 处理每行：
    • 每一行根据空格分割为不同的 token。
    • 根据 token 类型执行相应的处理函数，解析每个 token，并构建语法分析树。
  3. 生成语法分析树：
    • 按照不同 token 的语法结构，逐步构建语法分析树的各个部分，包括 state、speak、listen、branch、silence、default、calculate 等。
  4. 验证语法树：
    • 在生成语法分析树后，进行验证确保其结构的正确性，包括至少有一个 state、exit 列表、default 状态、silence 状态、listen 时间和 branch 状态是否有效。

interpreter.ts

​ 是执行脚本的模块，包含了与执行脚本相关的函数。这些函数用于解释和执行定义的语法树（AST），将语法树和运行环境传进去，获得回复和新的环境。

包含的函数：

  1. getVars:

    getVars 函数用于从语法树中提取变量名和对应的值，方便后续使用。

    • 用于从语法树中获取变量和对应的值。

  2. initEnv:

    initEnv 函数用于初始化运行环境，将当前状态设置为入口状态，初始化变量表。

    • 初始化运行环境，返回初始步骤和变量表。

  3. interpret:

    interpret 函数是核心执行函数，根据当前的状态、用户输入等信息，决定如何执行脚本，切换状态，并生成相应的文本回答。这个函数负责执行整个脚本的交互逻辑。

    • 执行脚本的主要函数。
    • 根据传入的语法树、运行环境、用户的回答等参数，确定执行逻辑并返回回应。
    • 根据当前状态，决定下一步的动作，并返回相应的文本回答。

Chat.vue

​ 提供了用户与脚本机器人交互的界面，包括消息的发送和接收，以及跳转到编辑页面的功能。

  1. 消息展示区域：
    • 通过 v-for 循环展示消息列表中的内容，分别显示用户和机器人的消息。
    • 采用不同的样式区分用户和机器人的消息，用户消息显示在右侧，机器人消息显示在左侧。
  2. 用户输入框：
    • 提供一个输入框，让用户输入消息。
    • 监听回车键事件 @keydown.enter.prevent，触发 sendMessage 函数发送消息。
  3. 跳转编辑页面按钮：
    • 提供了一个按钮，点击可以跳转到编辑页面。
  4. 消息发送和接收逻辑：
    • sendMessage 函数接收用户输入的消息，创建用户消息对象，并调用 onMessageWasSent 函数发送消息。
    • robotReply 函数接收用户消息，进行解释和处理，产生机器人回复，并调用 onMessageWasSent 函数发送机器人的回复消息。
    • 消息展示逻辑和样式设计使得用户和机器人的消息能够区分开，并以不同的方式展示。
  5. 与其他模块的交互：
    • 通过 DataStore 模块获取和存储全局信息，包括活动代码和语法树。
    • 使用 DataStore.userList 存储用户的信息、环境和消息列表，以便于保留用户聊天状态。
  6. 消息发送后的处理：
    • 当发送消息后，页面会自动滚动到最新的消息处，确保用户能够看到最新的聊天内容。
  7. 用户聊天结束处理：
    • 当聊天结束时，禁止用户继续输入消息，并显示提示信息。
    • 保存聊天信息和环境变量，以便下次聊天恢复上一次的状态。

Edit.vue

​ 编辑器和编辑器相关按钮的模块，可以可视化的编辑脚本和应用

  1. 编辑器界面：
    • 使用 Monaco Editor 作为代码编辑器，支持 Elixir 语言。
    • 提供代码编辑器的基本设置，如宽度、高度、主题等。
    • 通过监听 @change 事件，实时更新编辑后的代码内容。
  2. 按钮区域：
    • 包含了两个按钮，一个用于应用编辑后的脚本 (applyScript)，另一个用于重置为默认脚本 (reset)。
    • 按钮具有样式，并且在鼠标悬停时有颜色变化，提供了视觉反馈。
  3. 跳转按钮：
    • 提供了一个按钮，点击该按钮可以跳转到聊天页面。
  4. 交互和逻辑：
    • 当用户修改代码时，onChange 函数会更新 codeText 的内容。
    • applyScript 函数用于将用户编辑的脚本应用到应用程序中，通过比较当前活动的代码和用户编辑后的代码，决定是否更新活动的代码。
    • reset 函数用于将编辑器重置为默认脚本，恢复到初始状态。
    • 通过 localStorage 将用户编辑的代码保存在本地，以便在刷新页面后保持编辑的状态。
  5. 页面布局和样式：
    • 使用了 Flex 布局和样式调整，使界面布局更加美观和响应式。
    • 按钮样式设计使得按钮在悬停时有颜色变化，增加了交互性。

接口

程序间接口

函数调用关系：

模块间的接口：

  1. interface.ts:
    • 这个模块定义了各种接口，如 SPEAK、LISTEN、DEFAULT、SILENCE、BRANCH、HASHTABLE、VARS、AST、ENV、ANSWER 和 MESSAGE。
    • 这些接口定义了语法树的结构，运行环境的数据形式，以及不同模块之间交互所需的数据类型。
  2. interpreter.ts 和 parse.ts:
    • 这两个模块使用了 AST、VARS、ENV 和 ANSWER 这些接口类型。
    • interpreter.ts 导出了执行脚本的功能函数 interpret，它接受 AST、ENV 和用户回答作为输入，并返回 ANSWER 类型的响应。
    • parse.ts 导出了将代码解析成语法树的功能函数 parse 和验证语法树的函数 validate。这些函数接受字符串作为输入，并返回 AST 类型的语法分析树或者进行语法树验证。

模块间的交互流程：

  1. parse.ts 与 interpreter.ts:
    • parse.ts 中的 parse 函数接受代码字符串作为输入，并将其解析成 AST 类型的语法树。
    • 这个语法树可以被传递给 interpreter.ts 中的执行函数 interpret，以及 validate 函数。
    • interpret 函数通过接收 AST 和运行环境 ENV 来执行脚本，并生成相应的回答 ANSWER。
    • 也可以通过 validate 函数验证生成的 AST 语法树是否符合预期的规范。
  2. 模块间数据的传递:
    • interpreter.ts 中的执行函数 interpret 使用 AST 和 ENV 这些接口类型，通过解释器执行脚本并返回响应。
    • parse.ts 中的 parse 函数返回一个解析后的 AST，它可以被传递到其他模块中执行脚本。

人机接口

Chat.vue

  1. 交互方式
  • 输入框： 用户通过输入框输入文本消息，回车发送。
  • 消息展示区域： 以对话气泡的形式展示用户和机器人之间的对话，用户消息和机器人消息被不同的样式区分开来。
  • Edit 按钮： 跳转到编辑页面的按钮，允许用户编辑脚本或代码。
  1. 数据传递
  • messageList： 包含用户和机器人之间对话消息的列表。每条消息有作者、类型和数据内容等字段。
  • userInput： 用户在输入框中输入的文本数据，作为用户发送给机器人的消息。
  1. 接口设计
  • sendMessage(text: string): 处理用户发送消息的函数，将用户输入的消息构建成消息对象并加入消息列表。
  • robotReply(text: string, silence = false): 机器人回复消息的函数，根据用户消息进行解析，生成机器人的回复消息并加入消息列表。
  • onMessageWasSent(message: MESSAGE): 处理消息发送的函数，将消息对象加入到消息列表中并做出相应的处理。
  • gotoeditpage(): 处理跳转到编辑页面的函数，允许用户编辑脚本或代码。
  • 其他变量和状态值： 比如 tmp.User, tmp.messageList, tmp.env 等，用于记录用户信息、消息列表和当前运行环境。

Edit.vue

  1. 交互方式
  • Monaco Editor： 提供了一个代码编辑器，允许用户在编辑器中编辑脚本或代码。
  • 应用和重置按钮：用于用户应用或重置编辑器中的代码。
  • Chat 按钮： 返回聊天页面的按钮，允许用户从编辑页面返回聊天界面。
  1. 数据传递
  • codeText： 代表用户在编辑器中输入或修改的代码文本。
  • DataStore.activeCode： 用于存储当前活动的代码，用户可以编辑并应用。
  1. 接口设计
  • gotochatpage(): 处理跳转回聊天页面的函数。
  • onChange(code: string): 处理用户在编辑器中代码变化的函数，将用户输入的代码保存到 codeText 中。
  • applyScript(): 应用用户编辑的脚本或代码的函数，将 codeText 中的内容应用到 DataStore.activeCode 中。
  • reset(): 重置编辑器中代码为默认值的函数。
